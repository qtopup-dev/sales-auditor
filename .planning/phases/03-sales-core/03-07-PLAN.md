<objective>
Fix two first-mount/UX bugs in the Phase 3 sales sheet:

1. **virtualizer outerSize=0 on fresh SalesTable** — when the "No sales yet" empty state is visible and user clicks "Add Row", SalesTable mounts for the first time. The ResizeObserver hasn't measured the container yet so `outerSize=0`, which causes the virtualizer to render 0 items and the AddRowForm never appears. Fix: add `initialRect: { width: 0, height: 600 }` to `useVirtualizer` so it renders items in the first paint cycle.

2. **AsyncSelect value={null} visual reset** — in AddRowForm, both Product and MOP `<AsyncSelect>` have `value={null}` (hardcoded). After selecting a product, the combo immediately shows the placeholder again because the controlled `value` prop is always null. Fix: track each selected option in local state and bind it as the `value` prop.

These are both single-file changes with no architecture impact.
</objective>

**Why this fixes it:** `initialRect` is the react-virtual v3 mechanism for providing an assumed container size during the first render cycle, before the ResizeObserver fires. Without it, `outerSize=0` causes `getVirtualItems()` to return `[]` and nothing renders. With `{ height: 600 }`, the virtualizer assumes a 600px visible window and renders enough items to fill it (including the AddRowForm row at index 0).
