```typescript
  mopId: number;

  receiver: string;
  notes: string | null;
  status: SaleStatus;               // Soft-delete field (CLAUDE.md Rule 3)
  createdById: number;
  createdByUsername: string;        // Denormalized for display — avoid join on list view
  lastEditedById: number | null;
  lastEditedByUsername: string | null;
  createdAt: string;                // ISO 8601 UTC string
  updatedAt: string;
}
```

Note on `productNameSnapshot` and `mopNameSnapshot`: The Prisma schema stores these as `priceSnapshot` for price. The sale row also needs name snapshots for product and MOP names to satisfy CLAUDE.md Rule 4 (never join to products for display price on historical rows). The Prisma schema in plan 03 will need `productNameSnapshot` and `mopNameSnapshot` columns on the `Sale` model — cross-reference with plan 03 schema.
