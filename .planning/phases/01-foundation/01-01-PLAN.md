**docker-compose.yml** (repo root):
```yaml
version: '3.9'
services:
  mysql:
    image: mysql:8.4
    environment:
      MYSQL_ROOT_PASSWORD: root
      MYSQL_DATABASE: alejinput_db
    ports:
      - "3306:3306"
    volumes:
      - mysql_data:/var/lib/mysql
    command: >
      --default-time-zone=+00:00
      --default-authentication-plugin=mysql_native_password
    healthcheck:
      test: ["CMD", "mysqladmin", "ping", "-h", "localhost", "-uroot", "-proot"]
      interval: 10s
      timeout: 5s
      retries: 5
      start_period: 30s

volumes:
  mysql_data:
```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| .env file → process.env | DB credentials and session secret cross from disk to runtime here |
| docker-compose.yml → MySQL | MySQL root password defined in Docker env; only accessible on localhost:3306 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Information Disclosure | .env file | mitigate | `.env` added to `.gitignore`; `.env.example` committed instead; SESSION_SECRET labeled "replace before production" |
| T-01-02 | Information Disclosure | docker-compose.yml MySQL root password | accept | `MYSQL_ROOT_PASSWORD=root` is dev-only; never used in production; Docker binds to localhost:3306 only (no `0.0.0.0` exposure beyond loopback) |
| T-01-03 | Tampering | package.json supply chain | mitigate | All dependencies pinned to specific semver ranges; `^` prefix on non-React deps allows patch updates; React is pinned exact (`18.3.1`) to prevent accidental major version upgrade |
| T-01-04 | Information Disclosure | TypeScript strict: false risk | mitigate | `strict: true` enforced in root tsconfig; overriding packages cannot relax it without explicit `"strict": false` (D-04 non-negotiable) |
</threat_model>
