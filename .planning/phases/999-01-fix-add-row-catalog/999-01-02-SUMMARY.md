## Summary

Updated three frontend files to replace raw API calls to admin-only `/api/products` and `/api/mops` with React Query cache reads from the new `/api/catalog/*` endpoints. SalesPage pre-fetches on mount; AddRowForm and EditableCell read from cache synchronously — eliminating 403 errors for moderators and lag after void operations.

- **Modified:** `packages/frontend/src/pages/SalesPage.tsx` — added two cache-warming `useQuery` calls (`catalog-products`, `catalog-mops`) with `staleTime: 5 * 60 * 1000`. Results not used as props (D-08).
- **Modified:** `packages/frontend/src/components/sales/AddRowForm.tsx`:
  - Added `useQuery` to import; added `cachedProducts`/`cachedMops` queries; added `isCatalogLoading` guard
  - Replaced async `loadProducts`/`loadMops` with synchronous cache-filter functions returning `Promise.resolve(...)`

  - Added `useQuery` to import; added `cachedProducts`/`cachedMops` queries
