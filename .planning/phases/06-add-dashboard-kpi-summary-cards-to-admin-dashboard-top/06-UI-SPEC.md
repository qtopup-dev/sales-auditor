## Typography

Inherited from Phase 4. Two weights only — 400 and 600. Phase 6 adds one new role.

| Role | Size | Weight | Line Height | Tailwind | Phase 6 Usage |
|------|------|--------|-------------|----------|---------------|
| Body | 14px | 400 | 1.5 | `text-sm font-normal text-gray-900` | Inherited — not used in new KpiCard elements |
| Label | 14px | 400 | 1.4 | `text-sm font-normal text-gray-500` | KpiCard header label ("Transactions", "Profit", "Turnover") — matches StatCard label style exactly |
| KPI sub-label | 12px | 400 | 1.3 | `text-xs font-normal text-gray-400` | Sub-period labels inside KpiCard: "Today", "Yesterday", "This Month", "Last Month" |
| KPI value | 18px | 600 | 1.3 | `text-lg font-semibold text-gray-900` | Per-period values inside KpiCard (smaller than StatCard Display at 24px because 4 values share card space) |
| Heading | 20px | 600 | 1.3 | `text-xl font-semibold text-gray-900` | Inherited — "Dashboard" page title, unchanged |
| Display | 24px | 600 | 1.2 | `text-2xl font-semibold text-gray-900` | Inherited — existing StatCard values (Total Sales, Total Revenue), unchanged |

**No weight 500 (`font-medium`) anywhere in Phase 6.** All elements use 400 or 600.

**KPI value vs Display distinction:** KpiCard values use 18px/600 (text-lg) not 24px/600 (text-2xl). The smaller size is intentional — a single KpiCard card displays 4 values in a 2×2 grid and 24px would crowd the layout. The StatCard Display size at 24px remains reserved for the all-time totals (Total Sales, Total Revenue) which occupy their own single-value cards below the KPI section.

## Component Contract: KpiCard

**File:** `packages/frontend/src/components/admin/KpiCard.tsx`

**Status:** NEW — does not exist yet.

### Props Interface

```typescript
interface KpiPeriods {
  today: string | number;
  yesterday: string | number;
  thisMonth: string | number;
  lastMonth: string | number;
}

interface KpiCardProps {
  label: string;
  periods: KpiPeriods;
  loading?: boolean;
  isCurrency?: boolean;
}
```

### Visual Anatomy

```
[bg-white border border-gray-200 rounded-md p-6]
  [Card header label — text-sm font-normal text-gray-500 mb-4]
  [2×2 grid — grid grid-cols-2 gap-4]
    [Cell: Today]         [Cell: Yesterday]
    [Cell: This Month]    [Cell: Last Month]
```

Each period cell:
```
[div — flex flex-col]
  [p — text-xs font-normal text-gray-400 mb-1]   ← sub-period label
  [p — text-lg font-semibold text-gray-900]       ← value (or skeleton)
```

**Exact Tailwind for card shell:** `bg-white border border-gray-200 rounded-md p-6` — identical to StatCard.

**Sub-period label order in 2×2 grid:**
- Top-left: "Today"
- Top-right: "Yesterday"
- Bottom-left: "This Month"
- Bottom-right: "Last Month"

This order reads most-recent-first (Today, Yesterday) then current-period / prior-period (This Month, Last Month). No abbreviations — use full label text as written above.

### Value Display Rules

**Transactions (isCurrency = false):**
- Display value as-is: `String(periods.today)`, `String(periods.yesterday)`, etc.
- No prefix, no suffix.
- Example: `42`

**Profit / Turnover (isCurrency = true):**
- Prepend `₱` directly to value string: `"₱" + String(periods.today)`
- Value arrives from API as string (e.g., `"1250.00"`) — do NOT parse as float (CLAUDE.md Rule 6).
- Example: `₱1250.00`

### Loading State

When `loading = true`, render the 4 animate-pulse skeleton blocks in place of the period values. The card header label and sub-period labels are always rendered (static text), only the value slot is replaced by a skeleton.

```
[Card header label — static text, always rendered]
[2×2 grid]
  [Cell: Today]
    [p "Today" — text-xs text-gray-400 mb-1]
    [div — animate-pulse bg-gray-200 h-6 rounded w-16]   ← skeleton
  [Cell: Yesterday]
    [p "Yesterday" — text-xs text-gray-400 mb-1]
    [div — animate-pulse bg-gray-200 h-6 rounded w-16]   ← skeleton
  [Cell: This Month]
    [p "This Month" — text-xs text-gray-400 mb-1]
    [div — animate-pulse bg-gray-200 h-6 rounded w-20]   ← skeleton (wider for monthly values)
  [Cell: Last Month]
    [p "Last Month" — text-xs text-gray-400 mb-1]
    [div — animate-pulse bg-gray-200 h-6 rounded w-20]   ← skeleton
```

Skeleton height: `h-6` (24px) — matches the line height of the text-lg value it replaces.
Skeleton width for Today/Yesterday: `w-16` (64px).
Skeleton width for This Month/Last Month: `w-20` (80px) — slightly wider to suggest larger monthly totals.

### No Empty State

When the API returns 0 for any period, display `0` (Transactions) or `₱0.00` (Profit/Turnover). There is no empty/zero state requiring separate copy — 0 is a valid data value and self-explanatory.

### No Interaction

KpiCard is display-only. No click, hover (beyond default browser behavior), or keyboard interaction. No focus ring needed. No tooltip.

## Interaction Contracts

### KpiCard — No Interaction

KpiCard has no user interaction. It is a passive display component.

### KPI Data Loading (Inherited from summary query)

The KPI section shares the `summaryLoading` boolean from the existing `useQuery<AdminSummary>({ queryKey: ['admin-summary'] })`. When `summaryLoading` is true:
- All 3 KpiCards receive `loading={true}`
- All 4 value slots in each card show animate-pulse skeletons
- The existing StatCards also show their skeletons simultaneously (same loading flag)

When the query resolves, all cards populate together (single query, single state transition). No staggered reveal.

### KPI Data Error

If the `admin-summary` query fails, `summary` is `undefined`. KpiCard receives the fallback `?? { today: 0, ... }` default periods defined in DashboardPage JSX (see Layout section above). Values display as "0" / "₱0.00" — no error state copy on the KpiCard itself. The existing stats banner already has this behavior (shows "—" for total values on error); KpiCard shows "0"/"₱0.00" via the nullish coalescing fallback in the JSX props.
